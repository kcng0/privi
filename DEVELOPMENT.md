# Development Guide

How to stand up the toolchain and work on PrivateHeart Vault. Design/architecture
lives in [`docs/`](./docs/README.md); this file is purely about the dev
environment.

## Prerequisites

| Tool | Version | Notes |
|------|---------|-------|
| Flutter SDK | latest **stable** (≥ 3.24) | Pinned via [`.fvmrc`](./.fvmrc) → use FVM. |
| Dart | bundled with Flutter | — |
| Java (JDK) | 17+ | This machine has JDK 21 — fine. Needed for Android Gradle. |
| Android SDK | platform 34/35 + build-tools + cmdline-tools | Licenses must be accepted. |
| A device | Android 8.0 (API 26)+ | Real device recommended; **install VLC** to test external playback. |

> This repository currently ships the **Dart/config sources only**. The native
> `android/` scaffolding is generated by `flutter create` during bootstrap (see
> below) and is not hand-authored.

## Fastest path: one-shot installer

On a fresh Ubuntu/WSL2 machine, install the whole toolchain at once:

```bash
./scripts/install-toolchain.sh   # Flutter + Android SDK + licenses + PATH
source ~/.bashrc                 # pick up PATH
```

It is idempotent (safe to re-run) and does **not** touch the project. Gotchas
(especially connecting a device on WSL2), optional extras (FVM, proxies), and
troubleshooting live in **[docs/toolchain-setup.md](./docs/toolchain-setup.md)**.
Prefer to do it by hand? The manual steps are below.

---

## 1. Install Flutter (via FVM — recommended)

FVM keeps the Flutter version pinned per-project (`.fvmrc`).

```bash
# Install FVM (Dart pub or the install script)
dart pub global activate fvm      # or: curl -fsSL https://fvm.app/install.sh | bash

# From the repo root — installs the pinned Flutter and links .fvm/flutter_sdk
fvm install
fvm use stable
```

Prefer a plain global Flutter? Install from https://docs.flutter.dev/get-started
and skip FVM; the `Makefile`/scripts fall back to `flutter` automatically.

## 2. Install the Android SDK

Easiest via Android Studio (SDK Manager) or `sdkmanager` cmdline-tools:

```bash
sdkmanager "platform-tools" "platforms;android-35" "build-tools;35.0.0"
yes | sdkmanager --licenses
flutter config --android-sdk "$ANDROID_HOME"
flutter doctor            # resolve anything still red
```

## 3. Bootstrap the project

Generates native scaffolding, fetches deps, runs codegen — **safe to re-run**:

```bash
./scripts/bootstrap.sh
# or:  make bootstrap
```

What it does:
1. `flutter create --platforms=android --org com.privateheart .` to produce the
   `android/` project.
2. **Restores** our hand-authored `pubspec.yaml`, `analysis_options.yaml`,
   `lib/`, `test/`, etc. from git (flutter create would otherwise overwrite the
   template versions).
3. `flutter pub get` + `build_runner build`.

> ⚠️ Because step 2 restores files from git, **commit before running bootstrap**
> (the initial scaffold commit already covers this).

## 4. Everyday commands

```bash
make run       # launch on a connected device
make test      # run tests
make analyze   # static analysis (flutter_lints + riverpod_lint)
make format    # dart format lib test
make watch     # code generation in watch mode (drift + riverpod)
make gen       # one-shot code generation
make apk       # release APK for sideloading
make help      # list all targets
```

Without `make`, prefix with `fvm ` (or nothing) e.g. `fvm flutter test`.

## Code generation

Drift tables and Riverpod providers use `build_runner`. After changing any
`@DriftDatabase`, table, or `@riverpod` provider, run `make gen` (or keep
`make watch` running). Generated `*.g.dart` files are git-ignored and rebuilt.

## Dependency versions

`pubspec.yaml` pins **starting** constraints. They are not yet locked (no
`flutter` on the authoring machine). On first `flutter pub get`, a
`pubspec.lock` is produced — **commit it** (this is an app). If resolution
fails, run `flutter pub upgrade` and confirm packages on pub.dev. Riverpod is
**3.x** (codegen) and Drift/media packages move fast — verify APIs against
current docs when implementing.

## Branching & commits

- Default branch: **`main`**. Do feature work on branches; open PRs into `main`.
- CI ([`.github/workflows/ci.yaml`](./.github/workflows/ci.yaml)) runs format
  check, codegen, analyze, test, and a debug APK build on PRs.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `flutter: command not found` | Install Flutter / FVM (step 1); reopen shell. |
| Analyzer errors on `*.g.dart` | Run `make gen`; generated files are excluded from lints. |
| `flutter doctor` Android license issues | `flutter doctor --android-licenses`. |
| External player test fails | Ensure VLC is installed on the test device. |
| bootstrap overwrote a file | It restores owned files from git — make sure you committed first. |
